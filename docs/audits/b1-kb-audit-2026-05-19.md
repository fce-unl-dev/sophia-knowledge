# B.1 Auditoría inicial del KB — 2026-05-19

## Alcance

Auditoría inicial del repositorio vigente `fce-unl-dev/sophia-knowledge` contra `indice.json`, antes de implementar el scraper determinístico de cursos y la ampliación de páginas de estudiantes.

No se auditaron repositorios anteriores ni experimentos fuera de este repo.

## Fuente vigente

- Repositorio canónico: `fce-unl-dev/sophia-knowledge`
- Archivo de carga de Sophia: `indice.json`
- Regla vigente: Sophia carga únicamente los MD listados en `indice.json`.

## Hallazgos

### 1. Archivo faltante en el índice

Se detectó una entrada en `indice.json` que apuntaba a un archivo inexistente:

- `cursos/cursos-de-formacion-activos.md`

Impacto:

- Produce 404 al intentar cargar el MD.
- Duplica conceptualmente la información de los 22 MD individuales de cursos.
- Puede generar contradicciones si en el futuro se crea como documento agregado.

Acción aplicada en esta rama:

- Se eliminó la entrada de `indice.json`.
- Se subió `version` del índice a `4`.
- Se actualizó `lastUpdated` a `2026-05-19`.

### 2. Cursos de formación: modelo canónico

Decisión vigente:

- Mantener **1 MD por curso**.
- No usar un MD agregado de cursos activos.

Motivo:

- Reduce duplicación.
- Facilita diff humano por curso.
- Permite altas/bajas/cambios puntuales sin reescribir un documento monolítico.

### 3. Herramientas de scraper existentes

Existen herramientas en `/tools/scraper/` y workflow de sincronización, pero deben tratarse como automatización candidata o intento previo hasta completar B.2/B.3.

Acción aplicada en esta rama:

- La fuente `cursos-de-formacion` en `tools/scraper/sources.json` queda aislada con `strategy: "TBD"`.
- Se marca como `discovery-only` hasta implementar un scraper determinístico que genere o actualice un MD por curso.
- Esto evita que el pipeline actual intente generar `cursos/cursos-de-formacion-activos.md`.

### 4. README desactualizado

El README indicaba `/posgrados/ → 11 carreras`, pero el índice vigente contiene 12 entradas de posgrado.

Acción aplicada en esta rama:

- Se actualizó la estructura documentada.
- Se aclaró la regla de 1 MD por curso.
- Se documentó que `/tools/scraper/` no debe considerarse fuente publicada sin PR aprobado.
